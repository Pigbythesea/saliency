## Matrix V2 neural encoding

### What was computed

Each model/ROI cell selects one transformer block and one ridge alpha using an
inner validation split, refits the selected representation and ridge model on
the outer training set, and reports target-wise Pearson correlation on the
outer test set.

`full_neural_encoding.csv` contains:

`model`, `roi`, `subject_id`, `layer`, `metric`, `metric_scope`, `n_train`,
`n_test`, `num_items`, `num_targets`, `mean_score`, `median_score`,
`std_score`, `mean_noise_normalized_score`,
`median_noise_normalized_score`, `selected_ridge_alpha`, `selection_score`,
`selection_primary_score`, `selection_layer_matches`,
`selection_alpha_matches`, and `source`.

### How scores were produced

The implemented path in `src/hma/experiments/neural_alignment.py` is:

1. Import external features and require exact image-order equality with the
   neural dataset.
2. Split 9,841 images with seed 123 into 7,873 outer-train and 1,968 outer-test
   images.
3. Split the outer training set into 6,298 selection-train and 1,575
   selection-validation images.
4. For each candidate layer, flatten token by channel features.
5. Fit randomized PCA with 512 components on selection-train only and transform
   selection validation.
6. Fit closed-form ridge models for alpha values from `1e-3` through `1e7`.
7. Select the alpha maximizing the primary validation score. Ties within
   `1e-6` choose the smaller alpha.
8. Select the first layer whose score is more than `1e-6` better than the
   current best, so candidate order controls exact ties.
9. Refit PCA on all 7,873 outer-training images for the selected layer.
10. Refit ridge on the outer training set and evaluate on the held-out 1,968
    images.

The ridge model centers X and Y, solves
`(X'X + alpha I)^-1 X'Y`, and restores an intercept. The primary raw target
score is Pearson correlation between predicted and observed responses over
test images. Constant columns receive zero.

`mean_score` is the mean target-wise Pearson correlation. The code also
computes `r2_score_from_r` as squared Pearson correlation and
`prediction_r2` separately. Noise-normalized target scores are
`pearson_r^2 / noise_ceiling` for finite, positive noise ceilings. The
selection primary in these Matrix V2 configs is
`mean_noise_normalized_score`.

### What was selected or tuned

- Candidate layers: blocks 0, 3, 6, 9, and 11.
- PCA: 512 components, randomized solver, no whitening, seed 123.
- Outer train fraction: 0.8.
- Inner validation fraction: 0.2 of outer train.
- Ridge alphas: `1e-3` to `1e7`.
- Selected layer and alpha are recorded in each cell's
  `selection_artifact.json`.

### What was aggregated

`scripts/summarize_paper1_matrix_v2_full.py::_load_neural_rows` requires one
selected `encoding_scores.csv` row per model/ROI cell and copies it into the
12-row summary. It also checks that the selected layer and alpha agree with

## Matrix V2 representational geometry

### What was computed

`full_geometry.csv` contains 48 rows:
3 models by 4 ROIs by 4 methods. Its columns are `model`, `roi`, `subject_id`,
`layer`, `geometry_method`, `score`, `valid`, `status`,
`num_images_total`, `num_images_used`, `subset_seed`, `subset_size`, and
`source`.

Each cell uses the layer selected by neural encoding. Geometry is computed on
the final PCA representation produced after refitting PCA on the outer
training set and transforming all 9,841 images.

### Where it was computed and what data entered

Geometry is produced inside
`src/hma/experiments/neural_alignment.py::run_neural_alignment` using
`src/hma/neural/rsa.py::linear_cka` and `subset_rsa`. It consumes the selected
model feature layer, the ROI response matrix, and the same ordered 9,841 image
IDs used by the encoding cell.

### How scores were produced

Linear CKA:

- Columns of model features and neural responses are centered over images.
- Score:
  `||X'Y||_F^2 / sqrt(||X'X||_F^2 * ||Y'Y||_F^2)`.
- All 9,841 images are used.

Subset RSA:

- Subset sizes: 512, 1,024, and 2,048.
- Seeds: 123, 456, and 789.
- Images are sampled without replacement, then indices are sorted.
- Model and response RDMs use correlation distance:
  one minus row-centered cosine similarity.
- The score is Spearman correlation between upper-triangle RDM values.
- Each observed subset row has a response-permutation control with the same
  subset size and control seed.

Linear CKA is observed once per model/ROI cell and paired with
response-permutation controls at seeds 123, 456, and 789.

### What was selected or tuned

Geometry inherits the encoding-selected layer and final PCA fit. Subset sizes,
seeds, and control seeds are fixed config choices, not selected from geometry
performance.

### What was aggregated

The summary copies all valid per-cell geometry rows: 120 observed rows and 144
response-permutation rows. `full_model_summary.csv` uses observed rows only,
averages RSA across the three seeds, and then averages linear CKA and
2,048-image RSA by model.

### What is uncertain from the repo

The three subset seeds quantify subset variation but are not a confidence
interval. The controls diagnose response correspondence under permutation;
they do not provide subject- or model-population uncertainty.

## Matrix V2 behavioral fixation metrics

### What was computed

`full_behavior.csv` contains 9 rows:
3 models by 3 datasets. Its columns include dataset/scope/model metadata,
`num_items`, `fixation_protocol`, `saliency_method`, tie-correction fields,
corrected and raw versions of NSS, shuffled AUC, AUC Borji, AUC Judd, CC,
similarity, and KL, plus `source`.

### Where it was computed and what data entered

Operational maps are exported by
`scripts/export_external_routing_maps.py::export_routing_maps` and scored by
`src/hma/experiments/saliency_benchmark.py::run_saliency_benchmark`. Inputs
are the three 2,000-row behavior manifests, external resource artifacts,
fixation points or task points, and generated behavior configs.

### How scores were produced

`src/hma/experiments/saliency_benchmark.py::run_saliency_benchmark` loads one
precomputed operational routing map for each manifest row. Fixation points are
rounded from x/y to row/column coordinates, clipped to the map, and made
unique.

- NSS: z-score the prediction map and average at fixation coordinates;
  constant maps return zero.
- AUC Judd: compare fixation pixels with all non-fixation pixels using
  tie-aware ranks.
- AUC Borji: compare fixation pixels with random negative pixels for 100
  splits.
- Shuffled AUC: sample negatives from fixation coordinates belonging to rows
  whose `map_key` differs from the current row, for 100 splits.
- CC: Pearson correlation of flattened maps; constant maps return zero.
- Similarity: sum of pixelwise minima after nonnegative sum normalization.
- KL: target-to-prediction divergence after nonnegative sum normalization and
  epsilon protection.

Metric controls use seed 123, at most 256 positive fixations per image, and at
most 256 shuffled-pool points per image. SALICON and CAT2000 use `points`;
COCO-Search18 uses `task_points`. COCO density maps are generated with Gaussian
sigma 10.

Constant and numerically near-constant predictions are detected before metric
normalization. AUC Judd, AUC Borji, and shuffled AUC are defined as 0.5; NSS
and CC are defined as 0. Positive constant maps are treated as uniform
distributions for similarity and KL. Raw metric values and map-range
diagnostics remain in per-image provenance, while only defined values enter
aggregation.

The per-image rows are reduced to arithmetic means in
`aggregate_metrics.json`; there is no bootstrap in this stage and no
summary-stage uniform-map correction.

### What was selected or tuned

No model parameter is selected from behavioral performance. Metric randomness
uses seed 123, 100 AUC splits, and configured caps of 256 positive and 256
shuffled-pool points per image.

### What was aggregated

Each metric is averaged over the 2,000 benchmark rows in its cell.
`full_behavior.csv` emits one row per dataset/model and keeps raw and corrected
metrics.

### What is uncertain from the repo

The map-key audit resolves path collisions and repeated-trial identity. The
behavioral aggregates still contain no bootstrap or observer-clustered
uncertainty.

## Matrix V2 cross-axis and quadrant summaries

### What was computed

`matrix_v2_cross_axis_quadrants.csv` contains 72 rows:
2 behavior scopes by 3 neural axes by 4 ROIs by 3 models.

Behavior scopes are free viewing and task search. Neural axes are
noise-normalized encoding, linear CKA, and 2,048-image subset RSA.

### Where it was computed and what data entered

`scripts/summarize_paper1_matrix_v2_full.py::_build_model_summaries` consumes
the compact neural, geometry, behavior, and efficiency rows. No raw images or
target arrays enter this stage.

### How scores were produced

`_rank_composite` computes, for every complete three-model observation, the
fraction of the other two models beaten plus half credit for ties. KL is
reversed because lower is better. Scores are averaged across metrics and
datasets within the selected behavior scope.

`_observation_ranks` applies the same pairwise rank score to each model within
each ROI for a neural axis. Scores at least 0.5 are labeled `high`; lower
scores are labeled `low`. The method field explicitly records
`descriptive_panel_median_rank_n3`.

The file states its uncertainty as not estimated because it is a one-subject,
three-model deterministic panel.

### What was selected or tuned

The neural axes and behavior scopes are hard-coded. The observation threshold
is fixed at 0.5; aggregate model quadrants use the median three-model score.

### What was aggregated

`full_model_summary.csv` also averages encoding rank and geometry rank into
`neural_geometry_rank_score`, divides behavior and neural/geometry ranks by
realized GFLOPs, and assigns aggregate quadrants using the median of the three
model scores.

### What is uncertain from the repo

These classifications have no resampling uncertainty and are highly discrete
with only three models. `high` means a rank score of at least 0.5 within this
panel, not an external threshold.

### How saliency maps were produced

- Vanilla gradient:
  `src/hma/saliency/gradients.py::vanilla_gradient_saliency` takes the
  absolute input gradient, maxima over color channels, and min-max normalizes
  each map. The target is the model's argmax logit unless configured otherwise.
- Grad-CAM:
  `src/hma/saliency/gradcam.py::gradcam_saliency` averages gradients spatially
  at a named 4-D feature layer, weights activations, sums channels, applies
  ReLU, bilinearly upsamples, and min-max normalizes.
- Attention rollout:
  `src/hma/saliency/attention_rollout.py::compute_attention_rollout` fuses
  attention heads, optionally discards low attention, adds identity, row
  normalizes, and multiplies layer attention matrices. CLS-to-patch relevance
  is reshaped and upsampled.
- Transformer relevance:
  `src/hma/saliency/transformer_relevance.py` captures attention and gradients
  for the selected target logit, applies ReLU to gradient-weighted attention,
  fuses heads, adds residual connections, row normalizes, rolls matrices out,
  and extracts CLS-to-patch relevance. The implementation describes itself as
  scoped Chefer-style gradient-weighted attention, not full AttnLRP.
- DeepGaze references:
  `src/hma/saliency/precomputed.py::precomputed_map_saliency` loads stored
  arrays. CAT2000 and COCO configs use collision-safe `{map_key}.npy`; SALICON
  uses `{image_id}.npy`.
- Task prior:
  `coco_search18_task_prior` in `src/hma/saliency/baselines.py` builds
  target/task-conditioned fixation priors from the configured full training
  manifest with target+task, target-only, task-only, and global fallback
  buckets.

The same seven metric definitions described for Matrix V2 are applied.

### Cross-axis scoring and sensitivity

in `_static2000`. KL and other lower-is-better metrics are sign-reversed before
correlation.

For every dataset, metric, saliency method, neural scope, ROI/mean, and geometry
method, the code computes across-model:

- Spearman behavior versus raw encoding;
- Spearman behavior versus noise-normalized encoding;
- OLS slope/intercept/R-squared for those relationships;
- Spearman and OLS behavior versus geometry;
- Spearman encoding versus geometry.

At least three models are required. Sensitivity includes leave-one-model,

insufficient models.

### Uncertainty computation

- treats targets as bootstrap units;
- draws 2,000 bootstrap resamples with replacement;

subject, ROI, and method. They are summarized by mean, median, min, max, and

summarized by ROI, subject/method, and all-subject/method groups.

   audit passes all 9 cells with zero missing or unexpected keys/files.

## External validation handoff

An external methodology reviewer should validate these exact facts against the
relevant literature:

- The neural encoding pipeline flattens token features, fits randomized
  512-component PCA inside the training data, selects layer and ridge alpha on
  an inner validation split, refits on the outer training set, and reports
  target-wise held-out Pearson correlation.
- Its reported noise-normalized score is squared Pearson correlation divided
  by a positive finite target noise ceiling, averaged over valid targets.
- The Matrix V2 split is 7,873 train and 1,968 test images for `subj01`, with
  6,298/1,575 inner selection train/validation images.
- Linear CKA is computed from image-centered feature and response matrices
  using the Frobenius-norm formula on all available images.
- Subset RSA uses correlation-distance RDMs and Spearman correlation of their
  upper triangles on deterministic image subsets.
- Matrix V2 operational routing maps represent DynamicViT retained patches,
  ToMe inverse merge-group sizes, or a static full-token mask; they are not
  class-attribution maps.
- Behavioral metrics are NSS, shuffled AUC, AUC Borji, AUC Judd, CC,
  histogram-intersection similarity, and target-to-prediction KL.
- Matrix V2 static uniform-map AUC is defined as 0.5 in the metric layer before
  aggregation, with raw values retained as provenance.
- Matrix V2 behavioral artifacts are keyed by `map_key`, while neural artifacts
  retain dataset `image_id`; COCO repeated paths share maps but retain distinct
  row identities.
- Efficiency profiling uses batch 16, 20 warmups, five repeats of 100 batches,
  CUDA synchronization, and explicit fvcore diagnostics.
- Matrix V2 efficiency was recorded on one NVIDIA L40S environment; the
  repeated-run variance is not repeated-hardware uncertainty.
- Matrix V2 quadrant labels are within-panel pairwise rank classifications for
  three models and one subject, with a 0.5 high/low threshold and no estimated
  uncertainty; repaired outputs mark every row `diagnostic_only`.
- Older cross-axis correlations operate across four or six model points after
  joining model-matched behavioral and neural summaries; sensitivity removes

- V1 subject-robustness encoding uncertainty bootstraps target-wise
  DINOv2-minus-ResNet correlation margins, pooling targets across ROIs for
  subject summaries rather than using a hierarchical bootstrap.
